# AWS - Amazon Web Services 
1. Cloud Computing Platform by Amazon 
2. Used to design scalable, fault-tolerant, secure, and cost-efficient architectures
3. Provides on-demand IT resources over the internet
4. Pay-as-you-go pricing model

(For designing scalable, fault-tolerant, cost-efficient, cloud architectures)

## Services provided:
1. S3 (Data Storage )
2. EC2 (Compute Instances)
3. RDS (Databases)

## Key AWS Features:

1. On-demand self-service
- Provision resources without human interaction

2. Pay-as-you-go pricing
- Pay only for what you use
- No upfront hardware cost

3. Scalability & Elasticity
- Scale up or down based on demand
- Automatically or manually

4. High Availability
- Designed using Regions and Availability Zones
- Fault-tolerant architectures

5. Global Infrastructure
- Multiple Regions worldwide
- Each Region has multiple Availability Zones

6. Security
- Built-in security services
- Shared Responsibility Model
- Supports compliance standards

7. Managed Services
- AWS manages infrastructure
- User focuses on application & data

## AWS Global Infrastructure:

Region:
- A geographical area
- Example: us-east-1, ap-south-1

Availability Zone:
- One or more data centers
- Isolated from failures
- Low-latency connection between AZs

Edge Location:
- Used by CloudFront
- Content delivery (CDN)

## Shared Responsibility Model:

AWS is responsible for:
- Security OF the cloud
- Physical data centers
- Hardware
- Networking
- Hypervisor

Customer is responsible for:
- Security IN the cloud
- IAM (users, roles, policies)
- Data encryption
- OS & application configuration
- Network traffic protection
